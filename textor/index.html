WEAVE — 織って、磨いて、渡せる形に

文字起こしは、
入口にすぎない。

生の書き起こしは、そのままでは渡せません。Textor は読める日本語に整え、固有名詞を直し、話者を分け、議事録にまとめるところまでを、積み上げ式で引き受けます。各工程は同じ JSON を読み書きして重なっていくので、要らない工程は外すだけ。二次使用のお膳立て——字幕・検索・引用に渡せる状態にして手渡します。

01 — 織る・文字起こし

torch を入れずに、
Mac でも速い。

実行環境を見てエンジンを自動で選びます。Windows の NVIDIA GPU、Apple Silicon は GPU / Neural Engine、その他は CPU。巨大な torch は要りません。既定は高精度・高速のモデルで、中の文字起こしエンジンは差し替え可能——より良いものが出れば、名前も使い勝手も変えずに乗り換えられます。フォルダ一括は resume 対応で、中断しても続きから。

02 — 磨く・読める日本語に

誤変換を、
辞書で直す。

全半角の正規化、フィラー（えー・あのー）除去、かな数詞→数字、句読点の整形。そのうえで、現場の固有名詞・誤認識を辞書で一括補正——保護リストで誤爆を避け、どこを直したかの監査ログを残します。検索・置換（表記ゆれ無視・既定 dry-run・Undo 可）も。ここが「磨き上げ」の中心です。

正規化・フィラー・句読点辞書補正＋監査ログ検索・置換

03 — 渡す・分けて、まとめる

誰が話したかを分け、
議事録にまとめる。

話者分離（ベータ）で発言を人ごとに色分け。そのうえで議事録・SOAP などの完成ドキュメントをローカル LLM で生成します。要約はタイムコード根拠つきで、「どこでそう言ったか」を音声に戻って確かめられる。もちろん生成 LLM も端末の中で動きます。

話者色分け＋⚠要確認＋タイムコード → 議事録の根拠に
04 — 渡す・見える化と横断検索

低い信頼度を炙り出し、
過去も横断で探す。

各セグメントの信頼度を常に保存し、怪しい箇所だけ ⚠要確認 で可視化。音声同期・話者色分け・検索・議事録の根拠リンクを 1 枚のオフライン HTML ビューアにまとめられます（外部通信なし）。過去の書き起こしは日本語の部分一致で横断全文検索。すべて端末の中で完結します。

⚠ 要確認ハイライト1枚オフラインHTMLビューア横断全文検索
✓ビューア：音声クリックで該当箇所へ1枚HTML — 外部通信なし
✓「薬剤」を全案件から 7 件横断検索 — 用語ハイライト
⚠末尾の定型幻聴を自動で炙り出し要確認 — 低信頼箇所だけ表示

FIELDS & PRIVACY — 現場ごとに、送らずに

機密の声ほど、
クラウドに置きたくない。

議事録・診療記録・法務の打合せ。外に出た瞬間に問題になる音声だからこそ、便利なクラウド文字起こしに載せられない——そういう現場のための道具です。処理はすべてこの PC の中。音源も書き起こしも、どこにもアップロードしません。案件が終わったら音源ごと消せば、PII は端末にも残りません。現場ごとのパックで、用語も出力もその分野に合わせて整えます。

議事録

決定事項・動議・採決を抽出し、議事録テンプレへ。長時間の会議も、話者を分けて読める形に。

FIELD 02 — 医療

診療記録（SOAP）

診療の音声を SOAP の下書きに。PII は完全オフライン運用。作成支援であり、最終確認は医療者。

pack: 診療（medical）
FIELD 03 — 法務

尋問・打合せ記録

争点・主張・証言・合意を抽出。話者分離との併用を推奨。最終確認は弁護士・法務。

pack: 法務（legal）

現場パック＝「用語辞書／誤認識補正／出力テンプレ／抽出ルール」の4点セット。--pack で切り替えます。パックは増やせる設計です。
