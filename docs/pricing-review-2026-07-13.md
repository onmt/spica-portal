## 改訂案（catalog 正本・onmt/spica 側の変更が必要）

### 1. ココハワタシガ：セール窓を「早割」に移す【優先・次の申告期前に要決定】

- 現行: 定価 ¥1,200／半額 ¥600 は「確定申告シーズン等の本物の期間限定」（catalog notes）。
- 問題: 需要が2〜3月に集中する製品で申告期に半額を張ると、**年間実売の大半が ¥600 になり定価が形骸化**。
  比較対照価格（¥1,200）の販売実績が痩せ、二重価格表示の「相当期間の販売実績」要件（景表法・
  自ら catalog に明文化した原則）と衝突しかねない。ユーザーにも「申告期まで待てば半額」が学習される。
- 案①（推奨）: **セールは 11〜12月の「早割」に限定**し、申告期（2〜3月）は定価で売る。
  「早くレシート整理を始める人が得」という誠実な物語になり、定価の販売実績が需要ピークで積み上がる。
- 案②: 割引率を 50%→25%（¥900）に浅くする。 案③: 二段構えをやめ ¥800 均一。
- catalog 変更点: `receipts-yearly` の notes（セール条件）を案①または②の文言に改訂。

### 2. ffkit 商用ライセンス：¥5,000 → **¥9,800**（暫定のうちに）

- B2B の業務納品ツール。DVD/BD 納品1件の外注・人件費（数千〜数万円）を毎回浮かせる道具が
  ¥5,000 では1案件で回収でき過小。B2B では安すぎが「趣味のスクリプト」シグナルにもなる。
- LP は「価格は暫定・販売開始時に確定」と明記済みのため、**販売開始前の今が唯一の変え時**。
- catalog 変更点: `ffkit-commercial`（相当 SKU）price を 9800 に。LP `ffkit/index.html` の ¥5,000 を追従。
  （サイト側は catalog 改訂の後に変更する——3リポ価格整合の原則。今回は未変更。）

### 3. 文字起こし（whisper）：**¥4,800〜5,800 買い切り**の枠で新設

- MacWhisper 等の買い切り相場帯。クラウド文字起こしの月額（¥1,000超/月）への正面差別化。
  群の階段（¥0 → ¥1,200 → ¥3,000 → **¥5,000前後** → ¥14,800 → ¥59,800〜）の空き段を埋める。

### 4. 「ずっと使える」の負債整理：**次期メジャー版の有償アップグレード権を今のうちに規約へ予約**

- 買い切り＋永続ライセンスサーバー（90日更新）＋永続アップデート期待は、一度きりの収益で経常費を
  背負う構造。将来の値上げ・再課金の手段は「メジャー版の別ライセンス」しかなく、**後出しは
  「ずっと使える」の裏切りに見える**。
- EULA（雛形正本 = `onmt/spica` playbooks/eula-template.md）へ追加する文言案:
  > 「本ライセンスは、購入時のメジャーバージョンについて永続です。同一メジャーバージョン内の
  > アップデートは無償で提供します。次のメジャーバージョンは別ライセンスとなる場合がありますが、
  > その場合も購入済みのバージョンは使い続けられます（既存ユーザーには優待を予定）。」
- あわせて LP の「一度きり・ずっと使える」に注記を要検討（現メジャー版が対象であることを正直に）。

### 5. EULA の 0009 乖離の解消【⚠️・台帳起票済み】

- `legal/eula.html` §1「端末数の計数や、起動のたびのオンライン確認は行いません」・§5「収集・外部送信
  しません」は、ライセンス方式 0007–0009（Solo/Pro＝90日ごとオンライン鍵更新・機器ハッシュ送信・
  台数認証）および prompter LP「2台まで認証」と矛盾（EULA は 2026-06-28 版のまま）。
- 文面改訂には `docs/license-protocol.md`（正本）の精読が必要なため本セッションでは保留。
  **MoR 販売開始前に必須**。上記4の文言と同時に改訂するのが効率的。

### 6. 語り方の原則（値付けの防衛線）

- 「サーバー費ゼロ」を対外的な値付け根拠に使わない（0009 以降、ライセンスサーバーの恒久固定費がある）。
- アンカーは架空の定価でなく実在の代替コスト（レンタル ¥55,000/日・クラウド年額）で張る。
- セールは需要ピークに重ねない・50% を超えない・**年間実売の過半が定価**であることを毎年確認する。
- ローカルファーストは値引きの言い訳ではなくプレミアムの根拠として語る。

## 群としての値付け原則（正本候補・catalog notes へ転記推奨）

1. 買い切りが憲法。年度パスも「その年の買い切り」と言い切り、サブスクは作らない。永続の約束は
   現メジャー版に対して行い、次版の有償アップグレード権を規約に予約しておく。
2. 棚を守る: ¥0（実用無料）→ ¥1,200（一般）→ ¥3,000〜5,800（個人プロ）→ ¥14,800（現場・本命）→
   ¥59,800〜（法人）。新製品はこの棚の一段に置き、段を新設しない。
3. アンカーは実在の代替コストで張る。セールは需要ピークと重ねず、定価の実在性を守る。
4. ローカルファーストはプレミアムの根拠。安さで買われた瞬間「守るために引いた線」は
   「機能を諦めた線」に読み替えられる。
