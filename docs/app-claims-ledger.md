# アプリ主張台帳（app-claims-ledger）＋ 定期巡回の手順

サイト（spica-portal）の LP は、別リポにあるアプリ（spica-prompter・receipt_flutter＝ココハワタシガ・
ffkit・whisper）の仕様を語る。**サイトの「主張」とアプリの「実装」が別リポにあるため、放置すると乖離する**
（2026-07-11、サイト担当がココハワタシガの OCR/CSV 仕様を把握できていなかった事例が発端）。

この台帳は「サイトが各アプリについて言っている事実」を1行ずつ列挙し、各行に **正本（source of truth）の在り処**と
**最終確認日**を持たせる。pre-release-check の S1（主張の裏取り）を、公開直前の一発から**定期の巡回**へ格上げする
ための土台。

> **役割分担（2026-07-14）**: 全リポを横断する巡回の**本体は HQ に集約**した（`onmt/spica` の
> [`fleet/PATROL.md`](https://github.com/onmt/spica/blob/main/fleet/PATROL.md)＝艦隊巡回台帳）。
> 本ファイルは「**サイトが語ることの裏取り**」という一製品の仕事＝艦隊巡回台帳の "spica-portal 行" を担う。
> サイトに関わる乖離・新機能は、まず HQ の巡回で拾い、その反映先として本ファイルを更新する。

- **正本の原則**：価格・SKU は `onmt/spica` の `billing/catalog.yaml`。ココハワタシガのコピー正本は
  `onmt/receipt_flutter` の `docs/release/app_store_metadata.md`。機能・プライバシーの事実は各アプリの**実コード**。
- **書き方**：断言する主張だけを載せる。「未掲載だが事実」も載せてよい（新機能を LP に反映する候補になる）。
- **状態**：✅=実コード/正本で裏取り済み ／ ⚠️=要再確認・掲載と実態にズレ懸念 ／ ❓=リポ未接続で独立検証できず。

## 定期巡回の手順（このファイル＝チェックリスト）

巡回セッションは、この台帳を1行ずつ歩いて次を行う。**pre-release-check S1 を定期実行にしたもの**。

1. **対象リポを最新化**：spica-prompter・receipt_flutter・ffkit・whisper・spica を pull。
   （巡回セッションに各リポの**読み取りアクセス**が必要。無い行は状態を ❓ のままにし、その旨を報告する。）
2. **各行の再検証**：正本（実コード/catalog.yaml/メタデータ）を開き、主張がまだ真か確認。真なら「最終確認」日を更新。
3. **乖離検知**：主張と実装がズレていたら状態を ⚠️ にし、**サイト側の直し（LP 文言）または台帳の直し**を起票。
4. **新機能検知**：各アプリの `ROADMAP.md` / `DECISIONS.md` / リリースノート / `docs/release/` の差分を見て、
   **台帳に無い新しい事実**（今回の OCR/CSV のような）を洗い出し、行を追加。LP 反映候補として起票。
5. **価格整合**：`billing/catalog.yaml` と LP・特商法・terms・StoreKit の金額が3リポで一致しているか確認。
6. **記録**：巡回結果を `release-checks/PATROL-YYYY-MM-DD.md` にコミット。⚠️/新機能があれば issue 化 or 通知。

### 巡回頻度（基本: 月1回／現在: **週1回**＝更新ラッシュ中）

- 平常時は**月1回**を推奨（ズレ・新機能を早めに拾える／負荷も軽い）。四半期・リリース時・手動オンデマンドも可。
- **2026-07-11〜**: アプリ更新ラッシュ中につき **週1回（毎週月曜 9:00 JST・Routine `trig_01DWYDQAa5jhh9VhgkY4mXgZ`）**に増強。
  ラッシュが落ち着いたら月1へ戻す（Routine の cron を `0 0 1 * *` に変更するだけ）。
  変更が無い週は手順3〜5を省略して「変更なし」の PATROL 記録だけ残してよい。
- 自動化する場合はこの環境の **Routine（スケジュール実行）**で「別セッションを月次起動 → 上記手順 → 差分を issue 化＋
  スマホ通知」を組む。Routine セッションには**対象アプリ各リポをソースとして持たせる**設定が前提。
- 手動運用なら「アプリ巡回して」と指示 → このファイルを開いて手順1〜6を回す、で成立する。
