# DESIGN.md — spica-portal デザインシステム

> 現行実装（style.css / app.js / kokohawatashiga inline CSS / legacy.css）からの抽出。**正本はコード**であり、
> 本書はその言語化。乖離を見つけたらコードではなく本書を直す。ブランド上位規範は `onmt/spica` の
> `BRAND.md` / `brand/tokens.json`（NIGHT パレットの出所）。

## 1. テーマ — 「夜の現場の、静かな精度」

夜の撮影現場。照明が落ち、モニターの光だけがある空間で、**一字も揺れない道具**が静かに仕事をしている——
その空気を可視化する。派手さ・営業臭を排し、**実測値と構造で語る**（数値は必ず実測、バッジは実装状態に追従）。

- **基調**: ダーク（NIGHT パレット）。ほぼ黒の紺 `#0b0d14` の上に、星明かりのような淡い青白のテキスト。
- **アクセント**: スピカ青（1等星スピカ＝ブランドの星）。リンク・CTA・強調・Pro 系の演出はこの1系統のみ。
- **世界観の小道具**: 星空 canvas（トップ背景）・四芒星のブランドマーク・「読み線」を思わせる水平線。
- **語り口**: 断言は短く、根拠を添える。装飾のための装飾をしない。余白と行間で品位を出す。
- **例外**: ココハワタシガ LP のみ**ライトテーマ**（後述 §7。対象ユーザーが違うため意図的に分離）。

## 3. タイポグラフィ

### スタック（自前ホスト woff2・外部フォント禁止）

| `--serif` | IBM Plex Serif → ヒラギノ明朝ほか | **見出し専用**（h1/h2・締めの一文 .no-sub） |

## 4. レイアウト

- コンテナ `--container: 1140px`・左右 pad 32px。本文最大幅 `--measure: 38em`。
- 角丸: カード・表・図版 = **12px**／ボタン = 5px／バッジ(ピル) = 999px／小要素 = 4〜6px。
- ボーダーは 1px `--line`。強調したい枠だけ `--spica-400`（例: Pro プランカード）。
- セクションは縦に大きく呼吸させる（見出し前後 56px 級）。詰めて情報密度を上げない。
- 横スクロールは**ページでは禁止**。幅が出る要素（比較表）は `overflow-x:auto` のラッパー内のみ許可し、
  スマホでは次列が「覗く」幅設定にしてスクロール可能と分からせる。

### プランカード `.plan`
地 `--night-900`・枠 `--line`・pad 30/26。本命 `.pro` は枠 `--spica-400`＋右上リボン `.rec`。
リスト行は上ボーダー区切り・行頭 `✓`（mono・`--ok`）。価格は mono 1.9rem。

### 機能比較表 `.cmptbl`
`.cmpwrap`（枠12px・overflow-x:auto）内。thead は mono 11px 大文字、Pro 列は `.procol`（青8%地）。
グループ行 `.grp` は `--night-850` 地の mono 小ラベル。セル記号: `✓`=`--ok`／`14日`=`--warn`／`—`=`--night-600`。

### 検証台帳風リスト `.ledger .row`
glyph（✓/⚠/・）＋見出し＋説明＋状態バッジの4点セット。セキュリティ節の「構造で語る」形式。

### 図版（airgap 構造図・返しビュー図など）
実 UI スクショ（`assets/`・EXIF ゼロ）か、`role="img"`＋aria-label 付きの HTML/SVG 図。装飾イラストは使わない。

## 6. モーション

- **リビール `.rv`**: 初期 `opacity:0; translateY(18px)` → 交差で `.in` 付与（IntersectionObserver
  threshold .12 / rootMargin 下 -6%）。transition **0.7s** `--ease: cubic-bezier(.2,.7,.3,1)`、
  兄弟間ディレイ `i%6 × 60ms`（最大240ms）。**一度きり**（unobserve）。
- **星空 canvas `#sky`**: トップの背景のみ。低密度・低速の明滅。コンテンツの可読性より前に出ない。
- **ボタン**: hover で -1px 浮上のみ。バウンス・スケール・派手なイージング禁止。
- **デモ（プロンプター窓）**: 実機の挙動（等速スクロール・読み線）を模す目的のときだけ動かす。
- **ガードレール**: `prefers-reduced-motion: reduce` で全モーション停止（transition none・アニメ none・
  scroll-behavior auto）。**JS 無効でも本文が読める**よう `<noscript>` で `.rv` を可視化（各ページ head）。

## 7. 例外テーマ — ココハワタシガ LP（意図的な分離）

対象が IT に不慣れな一般ユーザー・高齢者のため、**夜テーマを使わない**（Koji 判断 2026-07-07）。

- アプリ実物のアイコン色 **#228486** を基準にしたライトテーマ・大きめ文字・専門用語回避。
- CSS はページ内インライン・**JS なし**・1ファイル完結（プライバシー主張と同型の「構造で安心」）。
- 法務2ページ（privacy/terms）の正本は `onmt/receipt_flutter` の `site/`。コピー正本は
  `docs/release/app_store_metadata.md`。
- 夜テーマの部品（.rv・星空・mono ラベル）を**持ち込まない**。逆輸入も禁止。

`legal/` は旧ポータル移植の `legacy.css`（独自変数体系）。**新規ページでの legacy.css 使用は禁止**、
夜テーマへの統一が将来課題（README 運用メモ参照）。

## 8. Guardrails（デザイン以前の憲法）

1. **外部リソースを一切読み込まない**。フォント・画像・JS・CSS すべて自前。CSP（`_headers`）が強制:
   `script-src 'self'`・`style-src 'self' 'unsafe-inline'`・`font-src 'self'`。CDN 追加は設計で却下。
2. **営業 CTA を置かない**（「ご相談ください」「お見積り」禁止。サイトの役割は名刺と入口。Koji 方針 2026-07-07）。
3. **数値・機能の主張は実測・実装のみ**。未実装機能は表・主張に載せない。裏取りは
   `docs/app-claims-ledger.md` → 各リポの実コード（fetch 後の origin/main）。
4. **価格・SKU の正本は `onmt/spica` `billing/catalog.yaml`**。サイト側で創作しない。
5. **公開＝ main への push の前に pre-release-check S1〜S5**（CLAUDE.md 参照・記録コミット必須）。
6. 画像は **EXIF/GPS ゼロ**で書き出す。スクショは開発版の旨をキャプションに明記。
7. アクセシビリティ最低線: h1 は各ページ1個・図版に aria-label・`:focus-visible` を消さない・
   reduced-motion と noscript のフォールバック維持・スマホ幅でページ横スクロールゼロ。
8. OGP は最小セット（og:image は #18 で整備予定）。meta description 必須。
