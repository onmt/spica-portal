ON-DEVICE — ぜんぶ、端末の中で

レシートの山を、
クラウドに頼らず片づける。

経費アプリの「当たり前」——画像をサーバーに送って OCR、データはクラウド保管——をやめました。読み取りも、AI も、集計も、すべてあなたの端末の中。プライバシーの心配ごとを、構造ごと取り除いた領収書アプリです。

Credo — 設計の考え方
  • 真実は、あなたの端末の中。レシート画像も記録も端末内のデータベースに保存。クラウドの障害や規約変更に、あなたの記録は影響されません。
  • 工業製品的信頼性。異常時もアプリ全体は落とさず縮退動作で継続するフェールセーフ設計。自動バックアップと復元性を最優先にしています。
  • 数字の出どころを、隠さない。OCR・学習・手入力・CSV——すべての数字に由来バッジが付き、確認すべき箇所がすぐわかります。
  • 会計ソフトと、共存する。置き換えるのはレシート管理だけ。お使いの会計ソフトはそのまま、必要な情報だけを CSV で渡します。
01 — On-device OCR + AI

読み取りも AI も、
端末の中で動く。

レシートを撮ると、OS の OCR と端末内の小型 LLM が日付・金額・店名・インボイス登録番号を抽出します。外部の OCR API には一切頼りません。候補から選ぶだけで、整う——使うほど店ごとの科目を学習し、次からの提案が賢くなります。学習データも、もちろん端末の中です。

仕訳に必要な数字だけ。

04 — 電帳法・インボイス

制度対応のチェックも、
端末の中で。

電子帳簿保存法・インボイス制度が求める日付・金額・発行者・登録番号の充足度を、1枚ごとに自動チェック。公共交通機関特例・自販機特例・少額取引特例のような「レシートが無くてもよい場合」の判定まで面倒を見ます。制度の細目を覚えるのは、アプリの仕事です。

充足度の自動チェック特例判定つき登録番号の自動抽出
